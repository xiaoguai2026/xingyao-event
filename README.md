# 星耀活动策划官网

## 项目说明

这是一个现代化的活动公司官网，包含：
- **前端展示网站**：响应式设计，深色现代风格
- **后台管理系统**：数据管理、统计功能

## 文件结构

```
xingyao-new/
├── index.html      # 前端网站
├── admin.html      # 后台管理
└── README.md       # 说明文档
```

## 部署方式

### 1. GitHub Pages 部署（推荐）

1. 在 GitHub 创建新仓库（如 `xingyao-event`）
2. 上传 `index.html` 和 `admin.html` 到仓库
3. 在仓库 Settings → Pages 中启用 GitHub Pages
4. 访问 `https://你的用户名.github.io/仓库名/`

### 2. 静态服务器部署

将文件上传到任何静态服务器：
- Nginx
- Apache
- Vercel / Netlify
- 阿里云 OSS / 腾讯云 COS

## 数据存储

本系统使用 **localStorage** 进行数据存储：
- 所有数据保存在浏览器本地
- 无需后端服务器
- 数据不会跨设备同步
- 清除浏览器数据会丢失所有内容

### 数据备份建议

定期在后台「系统设置」中导出数据备份。

## 自定义配置

### 修改公司信息

编辑 `index.html` 中的以下内容：
- 公司名称：搜索 "星耀" 替换
- 联系方式：修改电话、邮箱、地址
- 统计数据：修改 `.stat-number` 的 `data-target` 值

### 修改默认密码

首次登录后台后，在「系统设置」中修改密码。

### 添加初始案例

在后台「案例管理」中添加，或使用默认案例（已内置6个示例）。

- 移动端浏览器

## 注意事项

1. **微信分享**：如需微信分享显示缩略图，需配置 `og:image` 为可访问的图片URL
2. **数据统计**：统计功能依赖前端埋点，数据保存在访问者的浏览器中
3. **数据安全**：localStorage 数据可被用户查看和修改，不适合存储敏感信息
